# Теория к Лабораторной работе №3: Физическое проектирование и SQL

## 1. Физическое проектирование

**Физическое проектирование** — процесс реализации даталогической модели в конкретной СУБД.

### Основные задачи:
1.  **Выбор типов данных:** Оптимальный подбор типов (например, `integer` vs `bigint`, `varchar` vs `text`, `timestamp`).
2.  **Создание таблиц (DDL):** Написание скриптов `CREATE TABLE`.
3.  **Ограничения целостности (Constraints):**
    *   `PRIMARY KEY` (Первичный ключ) — уникальность и not null.
    *   `FOREIGN KEY` (Внешний ключ) — ссылочная целостность.
    *   `UNIQUE` — уникальность значений.
    *   `CHECK` — проверка условий (например, `age > 0`).
    *   `NOT NULL` — обязательность заполнения.
    *   `EXCLUDE` (PostgreSQL) — сложные ограничения (например, непересекающиеся периоды времени).

## 2. Язык SQL (Structured Query Language)

Стандартный язык для работы с реляционными базами данных.

### Группы операторов:

1.  **DDL (Data Definition Language)** — определение структуры данных.
    *   `CREATE`, `ALTER`, `DROP`, `TRUNCATE`.
2.  **DML (Data Manipulation Language)** — манипулирование данными.
    *   `INSERT`, `UPDATE`, `DELETE`, `SELECT`.
3.  **DCL (Data Control Language)** — управление правами доступа.
    *   `GRANT`, `REVOKE`.
4.  **TCL (Transaction Control Language)** — управление транзакциями.
    *   `COMMIT`, `ROLLBACK`, `SAVEPOINT`.

## 3. Индексы

**Индекс** — структура данных, повышающая скорость поиска информации в таблице за счет дополнительных затрат на запись и хранение.

*   **B-Tree:** Стандартный тип индекса, подходит для большинства задач (равенство, диапазоны).
*   **GiST/GIN:** Для сложных типов данных (геометрия, полнотекстовый поиск, массивы).

## 4. Миграции

**Миграция БД** — процесс изменения схемы базы данных (версионирование схемы). Позволяет синхронизировать состояние БД между разработчиками и средами (dev, test, prod).
