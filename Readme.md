# МОДЕЛИ ДАННЫХ и СИСТЕМЫ УПРАВЛЕНИЯ БАЗАМИ ДАННЫХ

## Общие требования

1. Для выполнения лабораторных работ вы можете использовать любую **реляционную СУБД** (например, PostgreSQL, MySQL).
2. Для сдачи лабораторной работы необходимо создать git-репозиторий, где `README.md` содержит ваши данные (Имя, Фамилия, Номер группы).

В данном курсе вы будете проектировать базу данных в контексте приложения, предметную область которого вы определите в первой лабораторной. Все последующие лабораторные работы будут дополнять предыдущие и расширять функционал вашей БД.
В конце курса вы получите реализованную базу данных, набор SQL-запросов и инструкций, готовых для применения в коде проекта.
Взаимодействие с базой данных в вашем проекте будет производиться исключительно посредством данного набора запросов. Проект должен также иметь интерфейс взаимодействия с пользователем.

Последняя лабораторная работа представляет собой готовый мини-проект, в котором в качестве базы данных используется реализованная вами БД.

# Лабораторная работа №1

### Необходимо:

1. Определить тему разрабатываемого проекта.
2. Определить функциональные требования к проекту.
3. Определить и обосновать перечень сущностей БД, удовлетворяющих функциональным требованиям:

   * Количество связанных сущностей — **не менее 10**
   * Промежуточная таблица many-to-many **не считается сущностью**, если не содержит дополнительной информации
   * Должны присутствовать **все виды связей**
4. Схематично изобразить **ненормализованную схему БД**.
5. Описать каждую сущность (имя поля, тип, ограничения, связи).

### Acceptance Criteria:

1. `README.md` содержит тему, ФИО, номер группы.
2. Определён, описан и закоммичен файл с функциональными требованиями. Минимальные требования:

   * Авторизация/аутентификация
   * Управление пользователями (CRUD)
   * Система ролей
   * Журналирование действий
3. Определён список таблиц.
4. Каждая таблица описана.
5. Есть схема предлагаемой инфологической модели БД.
6. Теоретические знания по теме.

# Лабораторная работа №2

### Необходимо:

1. Разработать даталогическую модель БД.
2. Привести БД к **третьей нормальной форме**.

   * При денормализации — уметь обосновать решение.

### Acceptance Criteria:

1. Разработана даталогическая модель.
2. БД приведена к 3НФ.
3. Готовность привести БД к BCNF, 4NF, 5NF, 6NF.
4. Теоретические знания по теме.

# Лабораторная работа №3

### Необходимо:

1. Разработать **физическую модель** БД (создать БД).
2. Наложить ограничения.
3. Заполнить тестовыми данными.
4. Создать пул SQL-запросов для простых операций.

### Acceptance Criteria:

1. Создана физическая модель с ограничениями:

   * Все таблицы и связи — **вручную**, без автогенерации.
   * Все SQL-скрипты сохранены.
   * Добавлены индексы, констрейнты и др.
2. БД заполнена тестовыми данными.
3. Составлен пул базовых SQL-запросов.
4. Готовность продемонстрировать работу запросов.
5. Готовность пояснить все операторы SQL.

# Лабораторная работа №4

### Необходимо:

1. Создать пул **сложных выборок**.
2. Пул запросов для создания представлений.
3. Пул запросов по группировкам.
4. Пул запросов для сложных операций над данными.
5. Проверить запросы на БД.

### Acceptance Criteria:

1. Сложные выборки:

   * Запросы с несколькими условиями
   * Вложенные запросы
   * Другие необходимые сложные выборки
2. Запросы для представлений, включая:

   * INNER, OUTER, FULL, CROSS, SELF JOIN
3. Запросы для группировок:

   * GROUP BY + агрегаты
   * PARTITION, PARTITION OVER + оконные функции
   * HAVING
   * UNION
4. Запросы сложных операций:

   * EXISTS
   * INSERT INTO SELECT
   * CASE
   * EXPLAIN
5. Не обязательно использовать *все* операторы.
6. Готовность объяснить все использованные операторы.
7. Теоретические знания по теме.

# Лабораторная работа №5

### Необходимо:

1. Создать пул триггеров.
2. Создать пул хранимых процедур.
3. Проверить их работу на БД.

### Acceptance Criteria:

1. Создан пул триггеров:

   * Примеры: пересчёт суммы заказа, логирование
   * Триггеры должны реализовывать **логику приложения**
   * Один триггер на логирование — **не засчитывается**
2. Пул хранимых процедур:

   * Процедуры для часто используемых действий
   * Должны храниться в БД и вызываться
3. Демонстрация работы процедур и триггеров.
4. Готовность создать новые.
5. Теоретические знания.

# Лабораторная работа №6

### Необходимо:

1. Создать приложение с пользовательским интерфейсом, использующее вашу БД как основную.

### Acceptance Criteria:

1. Приложение имеет пользовательский интерфейс:

   * Консоль: список команд
   * Web/desktop: интерфейс взаимодействия
2. Соответствие ТЗ (из ЛР1).
3. Используется БД, созданная в ЛР 1–5.
4. Взаимодействие с БД производится одним из способов:

   * Прямые SQL-запросы
   * Самописная ORM
   * ORM.raw_sql() или аналоги
     (**важно:** работа исключительно через SQL)
5. Используются запросы из ЛР 3–5.
6. Готовность редактировать код при сдаче.
7. Теоретические знания.
