# Функциональные требования к проекту "HomeFlip"

## 1. Общее описание
Проект "HomeFlip" представляет собой платформу для обмена жильем (Home Exchange). Пользователи могут размещать свои объекты недвижимости и искать варианты для обмена на время отпуска или путешествия.

## 2. Пользователи и Роли
Система поддерживает ролевую модель доступа (RBAC).
Основные роли:
* **Гость (Guest):** Неавторизованный пользователь.
* **Пользователь (User):** Авторизованный участник обмена.
* **Администратор (Admin):** Управляющий системой.

## 3. Функциональные возможности

### 3.1. Авторизация и Аутентификация
* Регистрация пользователей (Email, Yandex ID).
* Вход в систему.
* Управление профилем (ФИО, телефон).

### 3.2. Управление недвижимостью (Accommodations)
* Добавление объекта недвижимости (Описание, Адрес, Характеристики, Удобства).
* Загрузка фотографий.
* Редактирование информации об объекте.
* Управление статусом доступности (Календарь занятости).
* Модерация объектов (для Администраторов).

### 3.3. Поиск и Обмен
* Поиск жилья по параметрам (Город, Даты, Количество гостей, Удобства).
* Просмотр деталей объекта.
* Создание предложения об обмене (Exchange Offer).
* Принятие или отклонение предложения обмена.

### 3.4. Отзывы и Рейтинги
* Оставление отзывов на объекты после завершения обмена.
* Просмотр рейтинга и отзывов.

### 3.5. Администрирование
* Управление пользователями (Блокировка, назначение ролей).
* Управление справочниками (Страны, Города, Типы жилья, Удобства).
* Просмотр логов действий (Журналирование).

## 4. Нефункциональные требования
* Безопасность хранения данных пользователей.
* Валидация вводимых данных.
* Целостность данных (Foreign Keys, Constraints).
